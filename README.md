# 航空部件生命周期与维修管理系统

本项目是数据库系统课程项目，主题为 **航空部件生命周期与维修管理系统**。系统围绕航空部件从入库、安装、拆卸、更换、维修、飞行使用统计到最终退役的完整生命周期进行设计，重点展示数据库在业务规则控制、数据一致性维护、历史追溯和统计分析中的作用。

项目采用 **Vue 前端 + FastAPI 后端 + MySQL 数据库** 的三层结构。其中，前端负责页面展示和用户操作，后端负责接口组织和结果返回，数据库层负责核心业务规则校验、事务控制、状态流转、历史记录保护和复杂查询分析。

> 项目中的飞机、部件型号、寿命周期、维修记录和飞行日志均为课程模拟数据，不代表真实航空器参数或真实航空维修业务标准。

---

## 一、项目功能概述

系统主要实现以下功能：

* 飞机、部件类别、部件型号和部件实例管理
* 飞机安装位置管理
* 部件安装、拆卸与更换
* 部件维修计划与维修记录管理
* 飞行日志记录与部件飞行小时统计
* 部件寿命预警与维修周期预警
* 部件退役管理
* 部件完整生命周期时间轴查询
* 维修统计、退役原因统计和数据库健康检查
* 关键业务操作审计日志记录
* 非法操作拦截和事务回滚验证

系统不是简单的增删改查项目，而是通过数据库约束、触发器、存储过程、事务和视图共同维护业务逻辑，体现数据库设计在复杂业务系统中的核心作用。

## 三、系统架构

项目整体采用三层结构：

```text
用户操作
   ↓
Vue 前端展示层
   ↓ HTTP / JSON
FastAPI 后端接口层
   ↓ SQL 查询 / 存储过程调用
MySQL 数据库核心层
```

各层职责如下：

| 层次            | 主要职责                                   |
| ------------- | -------------------------------------- |
| Vue 前端展示层     | 提供部件管理、维修退役、统计追溯等页面，负责数据展示和用户交互        |
| FastAPI 后端接口层 | 提供统一 API，接收前端请求，组织参数，调用数据库对象，返回结果或错误信息 |
| MySQL 数据库核心层  | 保存核心数据，执行约束检查、触发器规则、存储过程事务和复杂查询        |

数据库层是本项目的核心。系统中的安装合法性、状态流转、退役限制、维修流程、飞行时间检查、历史记录保护等规则，主要在数据库层完成，而不是仅依赖前端提示或后端判断。

## 七、数据库设计亮点

### 1. 状态与历史分离

系统没有只用一个状态字段覆盖历史，而是通过多个历史记录表保存部件生命周期过程：

* `Component` 保存部件当前状态；
* `InstallationRecord` 保存安装与拆卸区间；
* `MaintenanceRecord` 保存实际维修过程；
* `MaintenancePlan` 保存维修计划安排；
* `FlightLog` 保存飞机飞行记录；
* `RetirementRecord` 保存退役事实；
* `AuditLog` 保存关键操作记录。

这种设计可以避免历史丢失，支持完整生命周期追溯。

### 2. 安装记录采用时间区间建模

`InstallationRecord` 通过 `install_time` 和 `uninstall_time` 表示部件在某架飞机某个位置上的安装区间。

* `uninstall_time IS NULL` 表示当前有效安装；
* `uninstall_time IS NOT NULL` 表示历史安装记录。

系统可以根据安装时间区间判断部件是否在某次飞行期间实际参与工作，从而推导部件飞行小时。

### 3. 数据库层负责核心业务规则

本项目将大量关键规则放在数据库层控制，例如：

* 退役部件不能再次安装；
* 已安装部件不能重复安装；
* 同一飞机同一安装位置不能同时安装多个部件；
* 部件类别必须匹配安装位置要求；
* 已退役状态为终态，不能恢复为可用；
* 历史安装记录、维修记录、飞行日志和审计日志不能被物理删除；
* 维修计划完成或取消后不能随意修改状态；
* 飞行日志时间不能重叠或不合理。

这样即使前端或后端出现遗漏，数据库仍能拦截非法数据。

### 4. 存储过程封装复杂事务

系统使用存储过程封装复杂业务流程：

* `sp_replace_component`：部件更换，在同一事务中完成旧件拆卸、新件安装和审计写入；
* `sp_retire_component`：部件退役，检查状态合法性后写入退役记录并更新部件状态；
* `sp_start_maintenance_plan`：启动维修计划，创建关联维修工单并更新计划状态；
* `sp_complete_maintenance`：完成维修，根据维修结果更新部件和维修计划状态。

这些操作涉及多个表，必须通过事务保证一致性。

### 5. 视图支持复杂查询与前端展示

系统通过视图封装常用查询逻辑，包括：

* 当前安装状态查询；
* 部件档案查询；
* 部件生命周期时间轴；
* 部件飞行小时统计；
* 寿命预警统计；
* 维修间隔分析；
* 维修计划详情；
* 退役原因统计；
* 审计日志详情；
* 数据库健康检查。

视图使前端和后端不需要重复编写复杂 SQL，也便于统一统计口径。

## 十、本地运行说明

### 1. 准备数据库

请先安装并启动 MySQL 8.0，然后按以下顺序执行数据库脚本：

```text
db/01_create_tables.sql
db/02_seed_data.sql
db/03_triggers.sql
db/04_procedures.sql
db/05_views_queries.sql
db/07_indexes.sql
```

执行完成后，数据库名为：

```text
aviation_maintenance
```

### 2. 配置后端环境变量

进入后端目录：

```powershell
cd backend
```

复制配置模板：

```powershell
Copy-Item .env.example .env
```

编辑 `.env`：

```dotenv
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=aviation_maintenance
```

注意：

* `.env.example` 可以提交到 Git；
* `.env` 存放本机真实数据库密码，不应提交到 Git。

### 3. 启动后端

创建虚拟环境并安装依赖：

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

启动 FastAPI：

```powershell
.\.venv\Scripts\python.exe -m uvicorn app.main:app --reload
```

启动成功后访问：

```text
http://127.0.0.1:8000
```

Swagger 文档：

```text
http://127.0.0.1:8000/docs
```

健康检查：

```text
http://127.0.0.1:8000/health
```

### 4. 启动前端

进入前端目录：

```powershell
cd aviation-frontend
```

安装依赖：

```powershell
npm.cmd install
```

启动开发环境：

```powershell
npm.cmd run dev
```

Vite 默认地址通常为：

```text
http://localhost:5173
```

如端口不同，请以终端输出为准。

生产构建验证：

```powershell
npm.cmd run build
```

本项目的主要特点如下：

1. **以数据库为核心**
   系统并不只依赖前端或后端判断业务是否合法，而是将关键规则下沉到数据库层，通过约束、触发器和存储过程保证数据一致性。

2. **保留完整生命周期历史**
   部件状态变化不会覆盖历史记录，安装、拆卸、维修、退役和审计事件都可以被追溯。

3. **事务处理复杂业务流程**
   更换、退役、维修完成和维修计划启动等操作涉及多个表，系统通过存储过程和事务保证操作的原子性。

4. **飞行小时统计具有可追溯来源**
   部件寿命预警不是简单依赖手工累计字段，而是根据飞行日志和安装时间区间推导，统计来源更加清晰。

5. **测试脚本覆盖正常流程和非法场景**
   项目不仅展示功能能执行，也展示数据库能够拒绝错误操作，并在失败时保证数据不被破坏。

## 十三、注意事项

* 本项目为课程项目，所有航空业务数据均为模拟数据。
* 数据库初始化脚本会重建数据库，执行前请确认目标数据库环境。
* `.env` 中包含本地数据库密码，不要提交到 Git。
* `node_modules/`、`.venv/`、`dist/`、`__pycache__/` 等目录不应提交。
* `06_test_illegal_ops.sql` 和 `08_test_procedures.sql` 包含测试写入和预期失败语句，不建议在正式演示前一次性全部执行。
* 若需要恢复演示数据，重新按初始化顺序执行 `01` 到 `07` 即可。

## 十四、项目总结

本项目围绕航空部件生命周期管理场景，完成了从数据库建模、数据初始化、业务规则控制、事务封装、复杂查询、后端接口到前端展示的完整实现。项目重点体现了数据库系统课程中的关系建模、完整性约束、触发器、存储过程、事务、视图、索引和测试验证等核心内容。

通过本系统，可以较完整地展示航空部件从基础定义、安装使用、维修处理、飞行统计到退役追溯的业务过程，并证明数据库层在保证业务规则正确性和数据一致性方面具有关键作用。
